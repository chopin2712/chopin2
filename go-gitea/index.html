Forget GitHub

  • SnowCode

    Read more posts by this author.

SnowCode

24 Nov 2019 • 2 min read

This title can sounds weird because this blog is hosted on GitHub but the only reason why I'm doing it is because Gitea doesn't support it. Maybe this is something that will be integrated later. If it is the case I will directly moves from GitHub to Gitea.

So let's see why.

GitHub is owned by Microsoft

GitHub has been owned by Microsoft even if it has been closed before. So, it means that you are sending all of your projects to Microsoft when publishing on GitHub.

Also, you are promoting one of their products. But one thing what is disgusting to me is that GitHub explains how to create open-source projects, but they are not open-source themselves.

It is a lack of honesty from them to do so.

GitHub is centralized

GitHub is centralized so if you want to host your files using the GitHub ecosystem without being on the central server you cannot use GitHub.

Alternatives

There are other alternatives too like GitLab or Gitea. GitLab can be quite complex and doesn't look like GitHub. If you just want to have a simple Git host instance you can use Gitea.

Git with a cup of tea

The Gitea server is easy to set up, you can do it within a minute! If you don't want to have your own Gitea server you can use the main Gitea server "gitea.com".

The official Gitea central server is free, not like GitLab who has different plans to be on the central instance.

Migrate from GitHub (or other platforms)

To migrate your projects from GitHub to your server. You can create a server or creating an account on gitea.com. Then click on "+" and New Migration.

Conclusion

So, to finish this post I would say to check alternatives to GitHub. I hope you will enjoy Gitea and you will create wonderful open-source projects!
