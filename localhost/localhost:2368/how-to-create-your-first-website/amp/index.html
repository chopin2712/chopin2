Today we will see how to build your first website in self-hosted using WordPress. I know that I made a post about “Ghost VS WordPress” but WordPress is still good in creation of pro websites.

Let's get started

Architecture of websites.

This is my view about the architecture of website.

Your website is powered by the engine who run on a server which is allowed on the internet by your ISP. And you ISP can be recognized into a DNS.

To get this more clear I will explain it with metaphors.

Your paper is in your binder which is in a box which is in a cupboard which is unlocked. To get access quickly to the cupboard you write on it things like “youtube.com” or “gitea.com”.

  1. The website: the files you got for example it could be WordPress.
  2. The engine: what I called “the Engine” is the way of starting these files we will see it later.
  3. The host: The computer or other material where the files are running.
  4. The ISP: Your internet service provider need to agree that anybody in the world can connect to your home through a specific door.
  5. The DNS: The DNS is the URL extension (like “.com” or “.org”). It gets a such quicker search than 172.217.11.78 instead of youtube.com.

All in one

If you are too lazy and you want your website fast, you can get a bundle of DNS, ISP, host and engine in one.

Some services allow you to rent servers and to install what you want on it, but because we are warriors we will do this, the hard way.

Let's get started!

Get the host.

First we need our host, remember that most the host is powerful, more your server will be protected against mass traffic.

You can choose the cheap not powerful option such as a raspberry PI (it is a $40 computer) or an old computer. If you got a bigger budget you can get a home server but this is more expensive than the Raspberry PI of course!

Also, I highly recommend you to use Linux on your device, first because Linux is nice and because this is a very common and easy way of doing things. Most server in the world runs Linux.

Fun fact: Your TV runs Linux too, if you got a smart TV. Also, Android is a Linux distribution for smaller screen.

Install the engine.

The “engine” we will use in this post is Docker.

A nice thing with Docker is that you can run multiple server at the same time!

To install Docker you can get the instructions on the official website.

If you are on Linux, you can get the official guide here: https://docs.docker.com/install/linux/docker-ce/ubuntu/.

Run WordPress!

To install WordPress on Docker we will use prebuild images.

Docker can be used in different way:

  • Build: A list of instructions that docker executes one by one
  • Hub: An image to run directly from your computer or from the cloud.
  • Compose: Running multiple images to make an environment.

We will use the Compose option, so let's go:

  1. Install Docker and run it
  2. Run the following commands: mkdir creates a directory and nano creates a file
mkdir wp
nano docker-compose.yml

3. Copy and paste the following code:

version: '3'

services:
  # Database
  db:
    image: mysql:5.7
    volumes:
      - db_data:/var/lib/mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wordpress
      MYSQL_PASSWORD: wordpress
    networks:
      - wpsite
  # phpmyadmin
  phpmyadmin:
    depends_on:
      - db
    image: phpmyadmin/phpmyadmin
    restart: always
    ports:
      - '8080:80'
    environment:
      PMA_HOST: db
      MYSQL_ROOT_PASSWORD: password 
    networks:
      - wpsite
  # Wordpress
  wordpress:
    depends_on:
      - db
    image: wordpress:latest
    ports:
      - '80:80'
    restart: always
    volumes: ['./:/var/www/html']
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_USER: wordpress
      WORDPRESS_DB_PASSWORD: wordpress
    networks:
      - wpsite
networks:
  wpsite:
volumes:
  db_data:

You can also change the different values to make your installation more secure!

The setup

Then you just need to configure your website using the simple automatic setup of WordPress. When you finished, you can see that your website has been created.

You can directly starting to write posts, pages, add themes or plugins. You can find lots of videos about it on the web. WordPress is a very popular choice to create websites.

The ISP authorization.

When you finished everything, you can make your server available to the world:

  1. Go on your ISP dashboard and login.
  2. Go into “Port Mapping” or “Security” section.
  3. Get the private IP of the host (“hostname -I”)
  4. Write the IP into the ISP like this:
    Protocol: TCP
    External Start,  External End and LAN: 80 (port of WordPress)
    Internal Host: The private IP of your host
    Description: The name you want.
  5. Confirm and search for your public IP address (type “what is my public IP address” on Google, Google should answer you.).

Now with this public IP, anybody can get access to your website.
